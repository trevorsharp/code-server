# HTML Document

Create an HTML document to present information clearly and effectively. Avoiding common pitfalls.

- The number one priority is information heiracrchy.
- Important information should be clearly presented and less-relevant information should be de-emphasized or omitted.
- The visual design should guide the viewer through the most important info.
- Simplify. Simplify. Simplify.
- Reduce any and all clutter.
- The goal is not to look super pretty but to effectively communicate the information.
- Don't add unnecessary subtext or decorative elements for the sake of aesthetics.
- Use a clear and consistent title that accurately reflects the content of the document.
- Break down into sections only as the content calls for it, avoiding unnecessary subdivisions.
- Implement light and dark mode support
- Contain all styles and scripts (if any) within the single HTML document itself.
- Be responsive to browser width.
- Use simple layouts.
- State each idea only once.
- Do not repeat conclusions in callouts, footers, highlighted text, or section summaries.
- Visual emphasis should only communicate new or highly important information.
- After drafting, remove any element whose deletion would not reduce understanding.
- Prefer deleting content over visually de-emphasizing unimportant content.
- Don't mention anything in the document that was only an idea generated during the creation process.
- The document should be understandable on its own merit and supported by the codebase.
